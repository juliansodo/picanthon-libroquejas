# OpenSpec

Estructura estándar usada:

- `project.md`: contexto y alcance del proyecto.
- `specs/<capability>/spec.md`: comportamiento consolidado vigente.
- `changes/<change-id>/proposal.md`: propuesta del cambio.
- `changes/<change-id>/design.md`: diseño técnico del cambio.
- `changes/<change-id>/tasks.md`: plan de ejecución verificable.
- `changes/<change-id>/specs/<capability>/spec.md`: delta de spec propuesto.
- `decisions/`: ADRs.
